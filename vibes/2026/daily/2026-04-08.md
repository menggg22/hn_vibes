## Personal Signal

*ml-infra / on-device ML / agent-systems — what's the trend and what sparks curiosity.*

**ml-infra & on-device ML**:
On-device ML is quietly maturing — Apple Silicon fine-tuning is becoming a first-class citizen, and the show of a Gemma 4 multimodal fine-tuner suggests the toolchain for local training (not just inference) is solidifying fast. Meanwhile GLM-5.1's long-horizon task focus points to the next infra challenge: maintaining coherent state over very long contexts.
- [Show HN: Gemma 4 Multimodal Fine-Tuner for Apple Silicon](https://news.ycombinator.com/item?id=47680309) — 138pts — what does the fine-tuning UX look like when it's local-first and multimodal? The gap between local inference and local training is closing.
- [GLM-5.1: Towards Long-Horizon Tasks](https://news.ycombinator.com/item?id=47677853) — 456pts — long-horizon capability is the new benchmark frontier; worth studying what architectural choices enable this.

**agent-systems**:
Agent infrastructure is hitting a trust/control crisis in real-time — the OpenClaw drama shows that developers building on top of agentic coding tools are exposed to unilateral platform changes, while Freestyle sandboxes and Scion's orchestration testbed represent attempts to build more durable, auditable agent environments. The meta-trend: builders want agent infra they own.
- [Launch HN: Freestyle – Sandboxes for Coding Agents](https://news.ycombinator.com/item?id=47663147) — 314pts — sandboxed execution environments for agents is the right abstraction; curious how they handle persistence and tool access boundaries.
- [Google open-sources experimental agent orchestration testbed Scion](https://news.ycombinator.com/item?id=47675213) — 168pts — a testbed from Google for orchestration is worth benchmarking against; useful signal on where Google thinks the hard problems are.
- [Show HN: Marimo pair – Reactive Python notebooks as environments for agents](https://news.ycombinator.com/item?id=47678844) — 24pts — reactive notebooks as agent environments is an underexplored idea; the notebook as an observable, replayable execution context could be genuinely useful for debugging agentic loops.
