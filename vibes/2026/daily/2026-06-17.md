## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The "local models are good now" story is the dominant signal — HN is collectively crossing an adoption threshold where on-device inference has become a realistic daily-driver, not just a hobbyist curiosity. The M1/M2 Mac as inference machine narrative is fully normalized.
- [Running local models is good now](https://news.ycombinator.com/item?id=48555993) — 950pts — worth reading for the specific benchmark claims and which models crossed which usability bars; the comments contain real practitioner setups

**agent-systems**: MCP (Model Context Protocol) is quietly becoming the plumbing layer for agent integrations, showing up in game engines (Claireon/Unreal) and dev tools; the pattern of "wrap existing tool surface with MCP" is accelerating faster than the underlying agent reasoning has improved.
- [Show HN: Ctx, save tokens by loading only the relevant tools](https://news.ycombinator.com/item?id=48559559) — 5pts — the token-cost problem of large tool registries is the unglamorous but critical bottleneck in production agent systems; this attacks it directly

**Deep dives**:
- [The time the x86 emulator team found code so bad they fixed it during emulation](https://news.ycombinator.com/item?id=48550693) — 476pts — a genuinely rare window into emulator internals and the philosophical question of whether a VM should silently fix bugs in the code it's running
- [Correlated randomness in Slay the Spire 2](https://news.ycombinator.com/item?id=48552844) — 271pts — deep game design analysis of how seeded RNG correlations create exploitable patterns; relevant to anyone building probabilistic systems
- [Formal Methods and the Future of Programming](https://news.ycombinator.com/item?id=48497067) — 68pts — lower score but the timing matters: as AI-generated code proliferates, formal verification as a correctness backstop is suddenly a practical not theoretical conversation
