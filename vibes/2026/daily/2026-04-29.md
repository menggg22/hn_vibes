## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The story today is shrinking the stack — AMD's Lemonade SDK dropping Electron to go 10× smaller, CJIT bringing C to JIT compilation, and WASM's stack-machine nuances all point toward a community obsessed with runtime efficiency as models get closer to the metal. Talkie's 13B vintage model from 1930 is the outlier curiosity: what does "vintage" mean for a language model?

- [Talkie: a 13B vintage language model from 1930](https://news.ycombinator.com/item?id=47927903) — 630pts — worth reading to understand what "period-accurate" training data does to model behavior and voice

**agent-systems**: The behavioral control layer is emerging as the real engineering problem — stop hooks being ignored (Claude 4.7), AGENTS.md quality mattering more than model choice, and a runtime proxy enforcing behavior all suggest the community is discovering that prompting is not enough and wants hard constraints. The TerminalBench OSS agent topping leaderboards shows the evaluation race is intensifying.

- [A good AGENTS.md is a model upgrade. A bad one is worse than no docs at all](https://news.ycombinator.com/item?id=47938417) — 96pts — the emerging "infrastructure as agent context" pattern; this framing will matter as codebases grow agent-native config files

**Deep dives**:
- [GitHub RCE Vulnerability: CVE-2026-3854 Breakdown](https://news.ycombinator.com/item?id=47936479) — 221pts 53c — with Ghostty leaving and GitHub availability incidents stacking up, understanding the actual attack surface is urgent reading today
