## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The conversation is shifting from "can we run models locally" to "can we run them cheaply and correctly" — token costs, distillation, and hardware cost comparisons are the new benchmarks. The Apple Silicon vs. OpenRouter thread makes this concrete: people are running real cost math on inference, not just vibes.

- [Apple Silicon costs more than OpenRouter](https://news.ycombinator.com/item?id=48168198) — 290pts — the crossover point where cloud inference beats owned hardware is moving fast and HN is doing the math in real time; worth reading the comment thread for actual numbers
- [CUDA Books](https://news.ycombinator.com/item?id=48168485) — 116pts — a curated reading list landing on the front page suggests a wave of engineers who want to go deeper than framework abstractions; demand signal for low-level GPU knowledge

**agent-systems**: The dominant theme is disillusionment with agent complexity combined with genuine excitement about narrow, well-scoped agent tools. Zerostack (pure Rust coding agent) and Semble (token-efficient search) both succeed by being opinionated and constrained rather than trying to be general.

- [Zerostack – A Unix-inspired coding agent written in pure Rust](https://news.ycombinator.com/item?id=48164287) — 537pts — "Unix-inspired" is doing heavy lifting here; HN rewards agents that respect composability and don't reinvent the shell
- [Show HN: Semble – Code search for agents that uses 98% fewer tokens than grep](https://news.ycombinator.com/item?id=48169874) — 123pts — points at a real emerging category: agent-aware dev tooling that optimizes for token budget, not just human readability
- [MCP Hello Page](https://news.ycombinator.com/item?id=48164294) — 126pts — MCP is quietly becoming infrastructure; a "hello page" existing suggests the protocol is past early-adopter phase

**Deep dives**:
- [Native all the way, until you need text](https://news.ycombinator.com/item?id=48168058) — 372pts, 249c — the text rendering problem in native apps is genuinely unsolved and this thread likely surfaces the real tradeoffs; essential reading if you're building anything that displays text outside a browser
