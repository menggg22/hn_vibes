## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The watermarking/detection arms race is heating up — reverse engineering SynthID shows researchers are already probing the reliability of AI-output detection before it becomes legally or commercially significant. Meanwhile hardware nostalgia (Z80 replacements, Nintendo DS programming) and the "one C file, no deps" aesthetic suggest a counter-current: builders who want compute they fully understand.

- [Reverse engineering Gemini's SynthID detection](https://news.ycombinator.com/item?id=47709130) — 99pts — if SynthID can be reverse-engineered this quickly, what does that mean for any regulatory framework that relies on watermarking?

**agent-systems**: The trust layer for agents is visibly cracking — the Vercel/Claude Code prompt-reading story and Claude's speaker-confusion bug both point to the same gap: agents lack robust identity and permission boundaries, and the tooling ecosystem around them (plugins, wrappers) is moving faster than the safety primitives. Research-driven agents (read before code) are one proposed answer.

- [The Vercel plugin on Claude Code wants to read your prompts](https://news.ycombinator.com/item?id=47704881) — 252pts — third-party plugin permission creep is the 2026 equivalent of browser extension overreach; this one has teeth because it's in a paid developer workflow

**Deep dives**:
- [Research-Driven Agents: When an agent reads before it codes](https://news.ycombinator.com/item?id=47706141) — architecturally interesting counter-proposal to "just vibe-code it"; worth reading alongside the Claude confusion thread to understand what guardrails actually look like
- [How Pizza Tycoon simulated traffic on a 25 MHz CPU](https://news.ycombinator.com/item?id=47703123) — a masterclass in constraint-driven design that has direct lessons for on-device ML inference budgets in 2026
- [Augmented Vertex Block Descent — WebGPU implementation](https://news.ycombinator.com/item?id=47702541) — AVBD in the browser via WebGPU is a meaningful milestone for in-browser physics/simulation; worth reading if you're tracking what WebGPU unlocks at the application layer
