## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The browser is becoming a serious ML runtime — Apple SHARP via ONNX Web, in-browser CAD, and client-side PDF generation all point to compute moving to the edge without native installs. Meanwhile, model cost anxiety (GPT-5.5 price hike) is pushing builders toward local/on-device alternatives.
- [Show HN: Apple's SHARP running in the browser via ONNX runtime web](https://news.ycombinator.com/item?id=47995037) — 182pts — Apple's own image super-resolution running client-side is a concrete milestone for on-device ML accessibility

**agent-systems**: The ecosystem is bifurcating: one camp is building agent *infrastructure* (sandboxing, versioning, review), while another is asking whether agents need fundamentally different primitives rather than wrappers around human tools. The UUID collision thread is a dark comedy cousin — systems designed for human error rates are meeting machine-speed operation.
- [Show HN: Git for AI Agents](https://news.ycombinator.com/item?id=48063548) — 92pts — the highest-signal fresh builder post today; frames the question of whether git's human-centric model is the right foundation
- [Natural Language Autoencoders: Turning Claude's Thoughts into Text](https://news.ycombinator.com/item?id=48052537) — 358pts — interpretability research that could matter enormously for agent debugging and auditability

**Deep dives**:
- [You gave me a u32. I gave you root. (io_uring ZCRX freelist LPE)](https://news.ycombinator.com/item?id=48067734) — a freelist integer confusion bug becoming a full LPE is a clean exploit narrative worth reading for the technique regardless of your kernel version
