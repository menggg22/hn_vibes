## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: On-device and offline inference is quietly becoming a serious workflow, not a toy — the flight LLM post and the Pico DSP firmware both point to capable compute running far from the cloud. Distributed training resilience (DiLoCo) is also getting attention as training scales away from single-cluster assumptions.

- [Running local LLMs offline on a ten-hour flight](https://news.ycombinator.com/item?id=47921064) — 107pts — practical field report on what actually works offline; useful signal for edge deployment realists
- [Fully Featured Audio DSP Firmware for the Raspberry Pi Pico](https://news.ycombinator.com/item?id=47901433) — 255pts — on-device signal processing at near-zero cost; points toward audio ML inference on microcontrollers
- [Decoupled DiLoCo: Resilient, Distributed AI Training at Scale](https://news.ycombinator.com/item?id=47924181) — 43pts — decoupling communication from compute in distributed training; worth watching as multi-cluster training becomes standard

**agent-systems**: The agent layer is maturing past demos — a benchmark-topping open-source terminal agent and a self-extending tool-building agent both shipped today, while the Prompt API signals browser-native agent primitives arriving in the platform layer.

- [Show HN: OSS Agent I built topped the TerminalBench on Gemini-3-flash-preview](https://news.ycombinator.com/item?id=47920787) — 289pts — open benchmarking of agents is finally happening; this is how the field gets honest

- [The Prompt API](https://news.ycombinator.com/item?id=47917026) — 253pts — browser-native prompt APIs mean agents don't need a server; architectural implications are large

**Deep dives**:
- ["Why not just use Lean?"](https://news.ycombinator.com/item?id=47922079) — 244pts 167c — formal verification keeps circling back as AI-generated code raises correctness stakes; the debate about when Lean is and isn't the answer is genuinely useful today
