## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The "LLMs as infrastructure components" conversation is maturing into failure-mode documentation — today's signal isn't capability demos but corruption reports and WebRTC edge cases. Real deployment friction is finally getting written up.
- [OpenAI's WebRTC problem](https://news.ycombinator.com/item?id=48051951) — 465pts — WebRTC's stateful, NAT-hostile nature is a real constraint on real-time voice/agent pipelines; this is the unglamorous infrastructure work nobody talks about

**agent-systems**: Delegation trust is the emerging fault line — two separate high-scoring stories today (LLM document corruption, Teaching Claude Why) both circle around the same question: what happens when you hand an agent authority over persistent state?
- [LLMs corrupt your documents when you delegate](https://news.ycombinator.com/item?id=48073246) — 334pts — empirical documentation of a failure mode that will only get worse as agentic file-system access becomes default behavior
- [Teaching Claude Why](https://news.ycombinator.com/item?id=48066592) — 246pts — Anthropic's attempt to solve delegation trust through value internalization rather than rule-following; worth reading alongside the corruption story

**Deep dives**:
- [You gave me a u32. I gave you root. (io_uring ZCRX freelist LPE)](https://news.ycombinator.com/item?id=48067734) — kernel exploit write-up that shows io_uring's zero-copy receive path is still yielding privilege escalation primitives; essential reading if you're running io_uring in production
