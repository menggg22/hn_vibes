**Top themes**: digital sovereignty, platform trust erosion, Linux/open-source renaissance, AI commercialization tension, dev tooling churn

**Vibe**: HN is in a mood of quiet defection — from GitHub, from Bambu Lab, from Claude, from US cloud, from GitHub again — the community is voting with its feet and its upvotes for anything that puts control back in the user's hands. ## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The big story is model compression reaching a tipping point — Needle's 26M distilled tool-calling model is the headline, but the broader signal is that capability is migrating down the size ladder fast enough that on-device inference is becoming a realistic default rather than a compromise. Meanwhile Python's GC revert in 3.14/3.15 is a reminder that runtime performance still matters enormously for inference workloads.
- [Show HN: Needle — Distilled Gemini Tool Calling into a 26M Model](https://news.ycombinator.com/item?id=48111896) — 633pts — worth exploring what the distillation dataset looked like and whether tool-calling generalizes or is task-specific at this size
- [Reverting the incremental GC in Python 3.14 and 3.15](https://news.ycombinator.com/item?id=48077924) — 190pts — GC behavior directly affects Python ML training loop latency; understanding why this was reverted matters for anyone running Python-native inference pipelines
- [Deterministic Fully-Static Whole-Binary Translation Without Heuristics](https://news.ycombinator.com/item?id=48117810) — 289pts — binary translation tech is adjacent to cross-architecture model deployment; worth reading for anyone thinking about running models on non-native hardware

**agent-systems**: Anthropic's pricing and access changes are forcing builders off Claude's API surface and onto open alternatives — you can see this in real time with the cluster of "observability," "guardrails," and "prompt capture" Show HNs today. The meta-trend is that agent infrastructure is being built defensively: people are wrapping agents in audit layers because they no longer trust the platform layer to be stable.
- [Tell HN: Don't use Claude Design, lost access to my projects after unsubscribing](https://news.ycombinator.com/item?id=48128003) — 113pts — concrete case study in platform lock-in risk for agent-dependent workflows; read the comments for the actual data loss mechanics
- [New Claude Code programmatic usage restrictions](https://news.ycombinator.com/item?id=48126438) — 35pts — the specific restrictions being imposed; essential reading before building anything on Claude Code as an agent substrate
- [Show HN: Statewright – Visual state machines that make AI agents reliable](https://news.ycombinator.com/item?id=48108778) — 112pts — state machine approach to agent reliability is worth comparing against the current "just prompt harder" school

**Deep dives**:
- [Linux gaming is faster because Windows APIs are becoming Linux kernel features](https://news.ycombinator.com/item?id=48087887) — the technical story here (Wine/Proton feedback loop into kernel) is a case study in how compatibility layers eventually reshape the thing they're compatible with; has direct implications for cross-platform ML runtimes
- [When "idle" isn't idle: how a Linux kernel optimization became a QUIC bug](https://news.ycombinator.com/item?id=48116064) — 154pts — the kind of layered debugging story that teaches you something about every layer of the stack; scheduler assumptions leaking into transport protocol behavior is a rare and illuminating failure mode
- [MacBook Neo Deep Dive: Benchmarks, Wafer Economics, and the 8GB Gamble](https://news.ycombinator.com/item?id=48125617) — 134pts 121c — "wafer economics" framing suggests this goes beyond benchmark porn into Apple's silicon supply chain bets; the 8GB floor decision is directly relevant to on-device ML viability
