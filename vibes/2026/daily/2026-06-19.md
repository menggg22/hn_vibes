## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The dominant signal today is a quiet but serious shift toward local inference as a production choice rather than a hobbyist curiosity — Qwen running locally, mistral.rs getting a skills API, and the top Ask HN all point to the same pressure. On-device ML is moving from "can it run" to "how do I integrate it into my workflow."
- [Local Qwen isn't a worse Opus, it's a different tool](https://news.ycombinator.com/item?id=48580209) — 445pts — The framing shift from "good enough" to "different capability profile" is the real story; worth reading the comments for concrete task-type breakdowns.

**agent-systems**: The A2A adoption question and the OSymandias runtime launch both reveal the same tension: agent *protocols* are proliferating faster than anyone is actually adopting them. Zero-Touch OAuth for MCP suggests the tooling layer is still being built out, not yet stable.
- [Zero-Touch OAuth for MCP](https://news.ycombinator.com/item?id=48592163) — 47pts — Reducing auth friction in MCP toolchains is a prerequisite for production agent deployments; the fact this is still a "solved problem" announcement tells you where the ecosystem is.
