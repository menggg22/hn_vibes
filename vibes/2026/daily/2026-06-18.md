## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: The on-device ML story is finally closing the gap with cloud — GLM-5.2 taking the open-weights lead on Artificial Analysis and the "running local models is good now" consensus post suggest the infrastructure has quietly crossed a threshold. The question is no longer "can you run it locally" but "what do you lose."
- [GLM-5.2 is the new leading open weights model on Artificial Analysis](https://news.ycombinator.com/item?id=48567759) — 757pts — worth understanding exactly which benchmarks it leads and whether the gap to proprietary models is now practically irrelevant for most coding tasks

**agent-systems**: Builder activity shows the field is fragmenting into two camps — Unix-native minimal harnesses (Relaymux, Zkit) versus managed runtimes (Mantyx, Agenthatch) — the same framework vs. library war that played out in web and then ML is now hitting agents. The "agentic coding deserves more than a chat box" framing crystallizes the dissatisfaction driving all of it.
- [Agentic coding deserves more than a chat box bolted onto VS Code](https://news.ycombinator.com/item?id=48571811) — 67pts — useful articulation of the UX failure mode that four separate Show HN builders are trying to solve today

**Deep dives**:
- [How we run Firecracker VMs inside EC2 and start browsers in less than 1s](https://news.ycombinator.com/item?id=48556561) — 179pts — directly relevant to anyone building browser-based agent sandboxes; the sub-1s cold start is the engineering number worth stealing
- [AI demands more engineering discipline. Not less](https://news.ycombinator.com/item?id=48570948) — 328pts — a counterweight to the "vibe coding is fine" discourse; read alongside the Fata spaced-repetition Show HN for a coherent argument about AI and skill maintenance
- [RFC 10008: The new HTTP Query Method](https://news.ycombinator.com/item?id=48568502) — 310pts — HTTP QUERY finally getting standardized matters for any agent or API system that needs to send structured query bodies without abusing GET/POST semantics
