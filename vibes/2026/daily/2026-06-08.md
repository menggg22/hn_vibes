## 🔭 Depth — ml-infra / on-device ML / agent-systems

*What's the trend and what sparks curiosity.*

**ml-infra & on-device ML**: Compression and efficiency are the dominant theme today — the community is looking hard at how to make inference cheaper rather than models bigger. The Biohub protein world model signals that domain-specific foundation models are arriving in earnest outside of language.

- [Speculative KV coding: losslessly compressing KV cache by up to ~4×](https://news.ycombinator.com/item?id=48400151) — 140pts — lossless 4× KV cache compression is the kind of result that pays for itself immediately in long-context workloads; worth understanding the mechanism
- [Biohub releases a world model of protein biology](https://news.ycombinator.com/item?id=48393252) — 148pts — world-model framing applied to biology rather than RL environments; the architectural choice here is the interesting question
- [Tokenomics: Quantifying Where Tokens Are Used in Agentic Software Engineering](https://news.ycombinator.com/item?id=48430923) — 165pts — empirical breakdown of token burn by task type in agentic loops; essential reading for anyone building agent infrastructure

**agent-systems**: Two threads today confirm the field is bifurcating between "agents as productivity tools for individuals" and "agents as infrastructure problems for enterprises." The Linux Claude thread is really an infrastructure-gap complaint in disguise.

- [Harness engineering: Leveraging Codex in an agent-first world](https://news.ycombinator.com/item?id=48416264) — 280pts — real production account of Codex in an agent-first engineering org; the workflow details are more useful than the headline
- [Show HN: Lathe – Use LLMs to learn a new domain, not skip past it](https://news.ycombinator.com/item?id=48433756) — 223pts — agents as pedagogical tutors rather than answer-dispensers; a framing that directly contradicts the dominant "skip the work" use pattern

**Deep dives**:
- [How's Linear so fast? A technical breakdown](https://news.ycombinator.com/item?id=48437609) — 257pts — local-first sync architecture explained in production context; rare honest breakdown of the engineering behind a beloved product's signature feel
- [Win16 Memory Management](https://news.ycombinator.com/item?id=48410844) — 126pts — understanding Windows memory constraints from 1985 reframes how much of modern OS design is inherited scar tissue
- [Sem: New primitive for code understanding – not LSPs, but entities on top of Git](https://news.ycombinator.com/item?id=48428475) — 168pts — proposes Git-entity layer as the right abstraction for code intelligence; worth reading if you're building any tooling that reasons about codebases
